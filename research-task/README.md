# 宏观经济与大类资产配置研究报告采集任务

## 任务定位

本任务独立服务于 `宏观经济与大类资产配置研究设计` 项目，目标是持续补充：

- 宏观经济周期研究报告
- 大类资产配置研究报告
- 经济周期识别框架研究
- 资产配置方法论研究
- 微信公众号文章与外部研究材料

这些内容后续将被拆分并并入主项目知识库，而不是简单堆积文件。

## 当前任务边界

本任务优先采集以下类型内容：

1. 经济周期理论
   - 美林时钟
   - 货币信用框架
   - 库存周期
   - 金融周期
   - 领先指标与转折点识别
   - 中长周期（朱格拉 / 康波）

2. 大类资产配置方法论
   - 风险平价 / 全天候
   - Black-Litterman
   - 宏观因子配置
   - 周期 + 因子 + 趋势的多层框架

3. 中国本土化研究
   - 中国宏观周期识别
   - 债股轮动
   - 宽货币 / 宽信用
   - 低通胀与资产配置

## 目录说明

- `reports/`
  - 存放下载后的报告原件
- `notes/`
  - 存放网页摘要、阅读笔记、拆分记录
- `sources_registry.md`
  - 来源清单与采集建议

## 与主项目的数据衔接

当前外部报告进入主项目时，不再直接把 PDF 链接硬塞进前端，而是先经过两层处理：

1. 自动抽取层
   - `scripts/build_external_reports_data.py`
   - 输出 `data/external_reports.json`
   - 负责提取摘要、关键图表页、匹配标签和新知识点
2. 人工校正层
   - `data/external_reports_review.json`
   - 负责修正自动抽取里不可靠的预览页、标题、状态和备注

最终再由主项目的 bundle 打包脚本把这些外部报告挂到：

- 理论框架页
- 宏观情景页
- 资产映射页
- 历史复盘页
- 配置方法论页

也就是说，这个采集任务的产出已经不是“报告仓库”，而是主系统的外部知识支持层。

## 已确认的关键限制

### 1. 微信搜狗可以作为重要补充来源

站点：
- https://weixin.sogou.com/

但目前自动化采集存在验证码和反爬限制，因此更适合作为：

- 手工检索入口
- 补充公众号深度文章
- 搜到后人工保存链接或截图

### 2. 外部网页来源适合先做“索引 + 摘要”

很多研究报告站点：

- 会有登录限制
- 会有跳转限制
- 不一定能直接稳定下载 PDF

因此当前最稳的流程是：

1. 先建立来源索引
2. 记录主题、作者、机构、时间、研究重点
3. 再根据价值高低决定是否单独下载原件

## 推荐工作流

1. 先补充高价值来源索引
2. 再挑重点报告下载原件
3. 原 PDF 进入 `外部研究报告/`
4. 运行 `scripts/build_external_reports_data.py` 生成结构化结果
5. 如自动抽取的关键图、摘要或预览页不准，再补 `data/external_reports_review.json`
6. 再拆分到主知识库的：
   - 理论框架
   - 宏观情景
   - 历史案例
   - 当前判断

## 当前维护原则

- 先修数据层，不先修前端样式
- 自动抽取不可靠时，优先在 `external_reports_review.json` 做人工校正
- 对结构损坏或文本质量很差的 PDF，宁可禁用自动预览，也不要挂错图
- 历史复盘页当前优先展示“文字复盘 + 来源 PDF”，而不是小缩略图
